Q1. What is the CIDR block assigned to the Lab VPC in this lab?
Explanation: The Lab VPC uses 10.0.0.0/16, providing ~65,536 private IPv4 addresses.
Q2. Which feature must be enabled so that EC2 instances receive DNS names automatically?
Explanation: Enable DNS Hostnames assigns friendly DNS names such as ec2-xx-xx-xx.compute.amazonaws.com.
Q3. The Public Subnet uses which IPv4 CIDR block?
Explanation: The public subnet occupies 10.0.0.0/24 within the 10.0.0.0/16 VPC range.
Q4. What setting makes a subnet public?
Explanation: A subnet becomes public when its route table sends 0.0.0.0/0 to an Internet Gateway.
Q5. Which component enables resources in the private subnet to reach the internet?
Explanation: A NAT Gateway lets private instances initiate outbound internet connections while remaining unreachable from the internet.
Q6. Which AWS service provides the virtual private network environment?
Explanation: Amazon VPC creates the isolated virtual network for your resources.
Q7. Why is a Bastion Host used?
Explanation: A Bastion Host (jump box) is an entry point to reach instances in private subnets.
Q8. Which subnet does the Bastion Host belong to?
Explanation: The Bastion must be in a public subnet so you can reach it from the internet.
Q9. Which subnet does the NAT Gateway reside in?
Explanation: The NAT Gateway must be in a public subnet and use an Elastic IP.
Q10. What route is added to the public route table for internet access?
Explanation: 0.0.0.0/0 represents all non-local traffic, which should route to the Internet Gateway.
Q11. What route is added to the private route table for outbound internet?
Explanation: Private subnets send internet-bound traffic to the NAT Gateway.
Q12. Why does the lab script set the password 'lab-password'?
Explanation: It enables password authentication so students can SSH from the Bastion into the private instance in an isolated lab.
Q13. Which command verifies that the private instance can reach the internet?
Explanation: Successful pings confirm outbound connectivity via the NAT Gateway.
Q14. If the NAT Gateway fails, what is the impact on the private subnet?
Explanation: Without the NAT Gateway, private instances cannot reach the internet.
Q15. What does the Internet Gateway do?
Explanation: The IGW provides a target for internet-bound routes and supports public IP connectivity.
Q16. Which AWS service can automate all the VPC creation steps?
Explanation: CloudFormation provisions resources declaratively using templates.
Q17. In the template, what property enables public IPs on instances launched in the public subnet?
Explanation: `MapPublicIpOnLaunch: true` on the subnet auto-assigns public IPv4 addresses.
Q18. Which resource type creates the 0.0.0.0/0 route to the IGW in CloudFormation?
Explanation: `AWS::EC2::Route` defines the specific route entry.
Q19. Which property ensures the NAT Gateway has a public IP?
Explanation: The NAT Gateway references `AllocationId: !GetAtt NatEIP.AllocationId` from an Elastic IP resource.
Q20. What is the main security benefit of using a private subnet?
Explanation: Private subnets block inbound internet access, reducing attack surface.
